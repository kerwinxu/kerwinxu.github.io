# 常用分层
1. entity : 实体类
1. dao    ：Data Access Object层，负责跟数据库交互的部分
1. dto    : 跟dao不同的是，这个是发给浏览器的，主要是控制发给浏览器的信息内容。
1. service ： 业务层
1. control : 控制器层，跟前端交互的

# 常用注解
1. jakarta.persistence.Entity  : 表明是一个实体类，跟数据库做对应的。
1. lombok.Data ， 会增加set,get函数
1. lombok.NoArgsConstructor ：会生成无参数的构造函数
1. lombok.AllArgsConstructor ： 会生成包括所有参数的构造函数
1. com.fasterxml.jackson.annotation.JsonIdentityInfo ： 主要是避免json出现死循环引用的
    1. @JsonIdentityInfo(generator = ObjectIdGenerators.PropertyGenerator.class, property = "id"		)
1. org.springframework.stereotype.Service; 表明是逻辑层
1. @Controller ： 控制层
1. @ResponseBody ： 相应的内容
1. @RestController ： 结合 @Controller和@ResponseBody，，其方法返回值会直接作为 HTTP 响应体内容（通常为 JSON 或 XML 格式），而无需额外的视图解析
1. @RequestMapping("/api") ： 可以指定请求的URL路径、HTTP方法、请求参数、头信息和媒体类型

# 注意
1. 保存或者修改数据的时候，如果有循环引用，比如保存设备类，其有部门属性，而部门属性中有设备列表，这时候保存的时候，最好将部门列表设置为null


# 跟vue集成部署
## vue 设置
vue的所有import ,如果是项目内的，都要根据@来定位，@表示项目目录，路由的话，选择hash，具体是router/index.js文件
```js
import { createRouter, createWebHistory, createWebHashHistory } from 'vue-router'
import global from '@/global'

const router = createRouter({
  history: createWebHashHistory(),    // 用hash吧，
  routes: global.router_config,
})

export default router
```
vite.config.js 文件
```
// https://vite.dev/config/
export default defineConfig({
  base: '/PositioningWeb/', // 设置为相对路径
  plugins: [vue(), vueDevTools()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
})

```
注意，spring不是放在根目录，是根目录下的一级子目录PositioningWeb，所以这里设置成'/PositioningWeb/',而在spring中，是如下设置的
```
server.servlet.context-path=/PositioningWeb
```

## spring配置

设置对应关系
```java
@Configuration
@Slf4j
public class WebConfig implements WebMvcConfigurer {


//    @Value("${file.staticAccessPath}")
//    private String staticAccessPath;
//
//    @Value("${file.uploadFolder}")
//    private String uploadFolder;

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // 获取当前项目运行目录的绝对路径
        String projectPath = System.getProperty("user.dir");
        String uploadPath = projectPath + "/uploads/";

        // 映射物理路径到虚拟URL路径
        registry.addResourceHandler("/uploads/**")  // 访问路径：http://域名/images/xxx.jpg
                .addResourceLocations("file:" + uploadPath) // 注意 file: 前缀
                .setCachePeriod(3600);
        registry.addResourceHandler("/static/**")
        		.addResourceLocations("classpath:/static/")
        		.setCachePeriod(3600);

        registry.addResourceHandler("/vue/**")
		.addResourceLocations("classpath:/static/vue/")
		.setCachePeriod(3600);
    }

}
```

设置验证
```java

@Configuration
public class SpringSecurityConfig {

	@Autowired
    private CustomUserDetailsService customUserDetailsService;
    @Autowired
    private JwtTokenProvider jwtTokenProvider;

    @Bean
    public static PasswordEncoder passwordEncoder(){
        return new BCryptPasswordEncoder();
    }

    @Bean
    SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {

        http// 由于使用的是JWT，我们这里不需要csrf
        .csrf(AbstractHttpConfigurer::disable)
        // 禁用跨域检测
//        .cors(AbstractHttpConfigurer:)
        // 禁用session
        .sessionManagement(configurer -> configurer.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        // 权限规则
        .authorizeHttpRequests((authorize) -> {

        	// 放行静态资源（所有非API的请求）
        	authorize.requestMatchers("/**").permitAll()
        	         .requestMatchers("/api/auth/**").permitAll()
        	         .requestMatchers(HttpMethod.OPTIONS,"/**").permitAll()
                   .requestMatchers("/api/**").authenticated();

                });
        // JWT 校验过滤器,添加过滤器才能真正的过滤
        http.addFilterBefore(new JwtAuthenticationFilter(jwtTokenProvider, customUserDetailsService), UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }

    @Bean
    public AuthenticationManager authenticationManager(AuthenticationConfiguration configuration) throws Exception {
        return configuration.getAuthenticationManager();
    }
}
```
